

```sh
# -x 删除所有文件，包括手动创建但是未提交的文件 (使用时需谨慎, 危险的)
# -X 仅删除 gitignore 文件中列出的文件
# 要删除至少给出下列三个选项之一
# -n 试运行, 实际上不要删除任何东西，只是展示将要做什么
# -f 强制执行的
# -i 交互式的
git clean -X -n
git clean -X -f

# -d 或 . 包括未跟踪的目录, 未跟踪的目录是指创建了一个新的目录，但是 git 并未提交过
# 如果一个目录已经被提交过了，那么该目录下的新建文件也是会被跟踪的，
# 然而如果在该目录下再新建一个目录，在该目录下新建的文件或目录将属于尚未被跟踪，
# 这时使用 -d 将指定并递归处理所有未跟踪的目录
# 如下将删除所有与工程无关文件，就和git push 再 clone 是一样的, 将得到一个干净整洁的目录
git clean -x -f .
```
